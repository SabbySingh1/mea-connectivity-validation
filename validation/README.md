# Connectivity Validation Scripts
Ben-Shalom Lab — Sabadnoor Singh, Summer 2025

Scripts for benchmarking spike train connectivity inference methods against known ground truth.
Every script prints its results to the terminal and saves them as a `.npz` file (precision, recall,
F1, and — where applicable — the full detected connectivity matrix and pair lists).

See `../connectivity_results.md` for the full write-up of every method, dataset, and outcome, with
plain-language explanations of what the numbers mean.

---

## Datasets referenced by these scripts

| Dataset | Source | Size | Notes |
|---------|--------|------|-------|
| **sim_cdkl5** | Donner et al. 2024 (spycon) | 123 neurons, 300s | Clean, non-bursty. ~8% connection density. Good benchmark. |
| **sim_burst** | Donner et al. 2024 (spycon) | 114 neurons, 257s | Bursty — 89% of spikes occur in network bursts. Closest analog to real CDKL5 data. |
| **Dale VAR** | Jan Balewski | 200 neurons, 300s | Statistical VAR model built for state estimation, not connectivity. 12.5% density but weak signal. |
| **trial_135** | Ben-Shalom Lab | 533 neurons, 300s | Abandoned as a benchmark — 39% ground-truth connection density is not biologically realistic. |

The current best synthetic benchmark for CDKL5-style data is the burst simulation generated by
`../simulation/generate_brian2_hdmea_burst.py` (see the root `README.md` for details) — it is matched
to real CDKL5 R59X firing statistics, not just a generic bursty network.

## Recommended / current scripts

These are the scripts that run cleanly against the CDKL5-matched simulation and accept
`--spikes` / `--conn` arguments to point at any dataset.

### glmcc_validate.py
**Method:** GLMCC (GLM-based Cross-Correlogram) — detects direct monosynaptic connections via a
sharp 1–5ms peak or trough in the cross-correlogram, fit with a smoothed background model.
**Use:** `python validation/glmcc_validate.py [--spikes ... --conn ...]`
**Result on sim_burst:** F1=0.195–0.038 depending on parameter set (see root README). Best-performing
method on bursty data so far, but still poor.

### dsttc_sccg_validate.py
**Method:** Directed STTC (co-firing within a coincidence window) or sCCG (smoothed cross-correlogram
with a Poisson null). Pass `sccg` or `dsttc` as the first argument.
**Use:** `python validation/dsttc_sccg_validate.py sccg` or `dsttc`
**Note:** DSTTC is O(N²·spikes²) — only practical below ~100–150 units. On 480+ unit datasets it can
run for many hours without finishing.
**Result:** F1=0.000 on bursty data for both — burst co-firing swamps the real signal.

### jitter_ccg_validate.py
**Method:** Jitter-corrected CCG. Computes the raw cross-correlogram, then subtracts the mean of many
"jittered" surrogate CCGs (each spike time randomly shifted ±25ms) to remove slow/burst-scale
correlation while preserving the fast 1–5ms synaptic peak.
**Result:** F1=0.408 on sim_cdkl5 (clean data), F1=0.022 on sim_burst — the 25ms jitter window isn't
enough to fully separate burst co-firing from the synaptic signal.

### te_validate.py
**Method:** Transfer Entropy — model-free measure of whether neuron A's spiking history improves
prediction of neuron B's next spike, beyond B's own history. Can catch non-linear dependencies that
CCG-based methods miss, but needs much more data to reach the same statistical power.
**Use:** `python validation/te_validate.py [--spikes ... --conn ...]`

### glmcc_sim_validate_original.py
**Method:** Same as `glmcc_validate.py`, but specifically for the original trial_135 dataset, with a
plotting step comparing ground truth vs. detected connections. Kept separate because trial_135's
ground-truth delays (200–1345ms) fall outside GLMCC's normal ±50ms search window, so results are
expected to be poor by design — this script exists to document that limitation, not as a working
benchmark.

## Earlier exploratory scripts (Dale VAR / trial_135 / novel methods)

These were run while investigating whether statistical/regression-based methods or burst-aware
modifications could succeed where CCG-based methods failed. All are kept for reference; none produced
a usable result.

### lasso_var_dale.py
**Method:** Lasso-VAR (lag-1) — L1-regularized regression predicting each neuron's activity from every
other neuron's activity one time step earlier.
**Dataset:** Dale VAR
**Result:** F1=0.064, Precision=0.119, Recall=0.044 — mathematically the right method for VAR-style
data, but the true connection weights are too weak relative to noise to recover reliably.

### stability_lasso_dale.py
**Method:** Bootstrap stability selection on top of Lasso-VAR — runs Lasso-VAR on 24 random 80%
subsamples of the data and only keeps connections that show up in ≥75% of the runs.
**Dataset:** Dale VAR
**Result:** F1=0.013 — the stability filter was too aggressive and discarded most true positives.

### granger_multilag_dale.py
**Method:** Granger causality (pairwise F-test comparing a full vs. reduced regression model) and a
multi-lag Lasso-VAR (lags 1, 2, and 3 combined, instead of just lag-1).
**Dataset:** Dale VAR
**Result:** Granger F1=0.078, multi-lag Lasso-VAR F1=0.056 (worse than lag-1 alone — extra lags added
noise, not signal).

### func_conn_dale_fdr.py
**Method:** Pearson correlation with Benjamini–Hochberg FDR correction, and STTC, both applied to
Dale VAR state 0 (the only state actually driven by network connectivity — see root README/results
doc for why).
**Dataset:** Dale VAR
**Result:** Pearson F1=0.000 (connected and unconnected pairs had statistically identical correlation),
STTC F1=0.131.

### func_conn_trial135.py
**Method:** Pearson correlation and STTC on trial_135.
**Dataset:** trial_135
**Result:** Pearson F1=0.362, STTC F1=0.401 — numerically fine but not meaningful, because trial_135's
39% ground-truth connection density is far denser than any real biological network, making these
scores uninterpretable as a measure of method quality.

### burst_excl_pearson.py
**Method:** Detects network-wide burst windows from the population firing rate, removes all spikes
inside those windows, then computes Pearson correlation on whatever spikes remain.
**Datasets:** sim_cdkl5, sim_burst
**Result:** F1=0.352 on sim_cdkl5 (moderate). F1=0.000 on sim_burst — 89% of all spikes occur inside
bursts, so almost nothing is left after exclusion.

### poprate_glm_burst.py — novel method attempt
**Method:** Adds the total population firing rate as an extra covariate in a Lasso-VAR model, on the
theory that it will absorb the shared burst drive and leave only genuine pairwise connectivity in the
remaining coefficients. Also tries Pearson correlation on the residuals after regressing out
population rate.
**Dataset:** sim_burst
**Result:** Lasso F1=0.036, Pearson-residuals F1=0.035 — both found most of the true connections
(recall ~0.83–0.87) but also flagged almost every other pair as connected (precision ~0.018), so the
burst drive was not adequately removed.

### sttc_cfp_burst.py
**Method:** STTC and CFP (Conditional Firing Probability), both evaluated against a proper null
distribution built from 1,000 circular-shift surrogates per pair (the standard approach used by the
MEA-NAP pipeline and by CFP's original 2007 paper on bursty cultured networks), rather than a fixed
threshold.
**Dataset:** sim_burst
**Result:** STTC F1=0.035 (Recall=1.00 — flagged every pair as connected, so the surrogate threshold
was too permissive here). CFP result not finalized due to runtime (13M+ surrogate evaluations in pure
Python — very slow).

### eann_run.py
**Method:** eANN — the ensemble neural network from Donner et al. 2024, which combines the outputs of
five base methods (CI, sCCG, dSTTC, GLMCC, GLMPP) through a trained neural net.
**Datasets:** sim_cdkl5, sim_burst
**Status:** Currently crashes due to an API mismatch with the installed spycon version
(`con_inf_dict` not being passed through correctly). Not fixed — the original paper already reports
eANN's performance on data similar to these two datasets, so re-running it here would only reproduce
published numbers rather than test something new. Kept in the repo for reference / future fixing.

## Key finding

No method tested here reliably detects connectivity in heavily bursty, organoid-like data. GLMCC is
the strongest performer on bursty simulations, but even it is far from reliable. The shared failure
mode across almost every method is that network-wide bursting makes every neuron pair look correlated,
and none of the corrections tried so far (burst exclusion, population-rate regression, jitter
correction, circular-shift surrogates) fully separate real synaptic signal from burst-driven
co-firing.
